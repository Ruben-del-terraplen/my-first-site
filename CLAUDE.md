## Current tech stack — STATIC HTML

The palette is monochromatic + 2 accents. **Do not introduce new colors.**

### Typography
Three fonts loaded from Google Fonts:
- **Instrument Serif** — display headings (hero titles, section titles, large quotes). Italic variant heavily used in the design.
- **Geist** — body text and UI
- **Geist Mono** — captions, meta labels, code-like accents (always uppercase with letter-spacing for editorial feel)

### Layout philosophy
- Generous whitespace, editorial pacing
- Magazine-style figure captions ("F. 01 — ...", "P. 01 — ...", "§ 01 — ...")
- Subtle reveal animations via `reveal.js` — opacity + translateY fades on scroll into view
- Big `[data-split]` text on heroes that reveal word-by-word
- The site has TWO motion patterns:
  1. `.reveal` — simple fade-up on scroll-into-view (used for prose, sections, metrics)
  2. `.reveal-img` — fancier "paper curtain" reveal (used for figures, project cards, image slices)

### Image slice pattern (`.cs-slices`)
This is the current preferred way to present case study images (added Phase 3, May 2026). Pattern:

```html
<div class="cs-slices">
  <figure class="cs-slice">
    <figcaption class="cs-slice-caption">
      <span><span class="num">F. 01</span> — Description · Subtitle</span>
      <strong>Right-aligned context</strong>
    </figcaption>
    <div class="cs-slice-figure reveal-img">
      <img src="images/case-N-slug/filename.webp" alt="..." loading="lazy" width="1440" height="HHHH" />
    </div>
  </figure>
  <!-- more slices -->
</div>
```

**Important:**
- Each slice gets a sequential figure number (F. 01, F. 02, ...)
- Set `loading="eager"` on the first hero slice, `loading="lazy"` on the rest
- Always include `width` and `height` attributes (matches the actual WebP dimensions) to prevent layout shift
- The hero slice uses its own `.cs-slices` wrapper with `style="margin-top: 48px;"` to sit right after the meta info
- The "design walkthrough" middle slices share one `.cs-slices` wrapper
- The footer/closing slice uses `class="cs-slice is-closing"` modifier (narrower, max-width 80%)
- Captions use Geist Mono, uppercase, letter-spacing 0.08em

### Video slice pattern (`.cs-video`)
Use video instead of a static image when the design's value is in motion — interactive flows, sticky behaviors, scroll interactions, transitions. The component sits inside a `.cs-slices` wrapper and reuses `.cs-slice-caption` for captions.

```html
<div class="cs-slices" style="margin-top: 24px;">
  <figure class="cs-slice cs-video">
    <figcaption class="cs-slice-caption">
      <span><span class="num">F. 01</span> — Description · Subtitle</span>
      <strong>Right-aligned context</strong>
    </figcaption>
    <div class="cs-video-figure reveal-img">
      <video
        src="images/case-N-slug/filename.mp4"
        poster="images/case-N-slug/filename-poster.jpg"
        autoplay muted loop playsinline
        preload="metadata"
        style="aspect-ratio: WIDTH / HEIGHT;"
        aria-label="Descriptive label"
      ></video>
    </div>
  </figure>
</div>
```

**Required attributes:**
- `autoplay` + `muted` + `playsinline` — this combo allows iOS to autoplay inline
- `loop` — continuous playback
- `preload="metadata"` — don't preload the whole file, just enough for dimensions
- `poster` — still frame, also acts as fallback and shows for reduced-motion users
- `aria-label` — accessibility
- `style="aspect-ratio: W / H;"` — prevents layout shift (use actual video dimensions)

**File specs:**
- **Codec:** H.264 (MP4 container), no audio (`-an`)
- **Max width:** 1440px (same as images)
- **Quality:** CRF 26-28, target under 5MB
- **Pixel format:** `yuv420p` for max browser compatibility
- **Faststart:** Always set `-movflags +faststart` for web playback
- **Poster image:** JPEG, extracted from ~1 second in, same dimensions as video

**Compression command:**
```
ffmpeg -i source.mp4 -an -c:v libx264 -crf 26 -preset slow -pix_fmt yuv420p -movflags +faststart output.mp4
ffmpeg -ss 1 -i source.mp4 -frames:v 1 -q:v 4 poster.jpg
```

**Reduced-motion handling:** `reveal.js` checks `prefers-reduced-motion: reduce` on load and pauses all `.cs-video-figure video[autoplay]` elements, letting the poster image display instead.

**Folder convention:** Same as images — files live in `images/case-N-slug/`.

## Image conventions

### Folder structure
- One folder per case study: `images/case-{N}-{slug}/`
- Sequential filenames with slug for clarity: `casino-01-hero.webp`, `casino-02-deals-slots.webp`, etc.

### Format & dimensions
- **Format:** WebP only (PNG/JPG fallbacks not needed — WebP has 95%+ browser support)
- **Width:** 1440px (the design width — displays sharply at retina on desktop, scales down on mobile)
- **Quality:** 82 (good balance of quality vs. file size for screenshots)
- **Target file size:** Under 350KB per image. Anything over 500KB needs more compression.

### Compression workflow
Source PNGs from Figma are typically 4-5MB at 2160px wide. To compress:

1. Resize to 1440px wide (preserve aspect ratio)
2. Convert to WebP at quality 82
3. Verify under 350KB

Tools: `squoosh.app` (manual, browser-based) or Pillow/ImageMagick (scriptable).

## Development conventions

### Commits
- Always commit and push after a working change. Don't pile up uncommitted work.
- Conventional commit prefixes: `feat:`, `fix:`, `style:`, `docs:`, `refactor:`, `chore:`
- Examples: `feat: add tournament case study slices`, `style: cap slice width at 1280px`, `fix: correct broken link in casino case study`

### Workflow per change
1. Make the change
2. Open the affected HTML file in a browser locally to verify (or use `npx serve .`)
3. Spot-check on mobile viewport (Chrome DevTools, Cmd/Ctrl + Shift + M)
4. Commit with a clear message
5. Push
6. Vercel auto-deploys in ~30 seconds
7. Verify live URL

### Things NOT to do
- ❌ Don't introduce a build step, framework, or `package.json` — this is a static site by design
- ❌ Don't install npm dependencies
- ❌ Don't add tracking/analytics without asking
- ❌ Don't recreate decorative motion (parallax, scroll-jacking, etc.) — site uses calm reveals only
- ❌ Don't fabricate content (employer history, metrics, project details) — all content comes from Figma or from Ruben directly
- ❌ Don't add new fonts or colors without asking
- ❌ Don't restructure the file tree without updating this CLAUDE.md
- ❌ Don't use scroll-within-scroll for image presentation — we deliberately chose stacked slices
